Section 2

RPC 入口 vs 内部循环

本文最重要的一条约定：recv_data_event()、accept_data_event()、 recv_loop() 三者不是重复，分别对应远程入口、本地桥接、常驻消费者。

RPC INGRESS

StageActor.recv_data_event()

Ray RPC handler · 一次 RPC 一次

  • 由别的 actor / API server 通过 .remote() 调用
  • 只做 metadata 校验、轻量状态更新
  • 调用本地 StageRuntime.accept_*()
  • 立刻返回 Ack
  • 不读 payload，不跑 batch，不直接 forward
LOCAL BRIDGE

StageRuntime.accept_data_event()

普通 Python 方法 · 不出现在远程接口里

  • 把 RPC handler 收到的消息交给 runtime 管理
  • metadata-only：把 DataEvent handle 放入 recv_buffer
  • 不长期运行，不阻塞 RPC 返回
  • 不读 payload body
INTERNAL LOOP

StageRuntime.recv_loop()

asyncio task / thread · 常驻运行

  • actor 启动后开启，一直循环
  • 从 recv_buffer 取事件
  • 调 transport.read_payload() 真正搬数据
  • 过滤 stale epoch、检查 credit
  • 写入 input_queue 喂给 ComputeLoop

三条 Loop 的分工

RecvLoopin StageRuntime
  • recv_buffer.get()
  • transport.read_payload()
  • filter stale epoch / dedup
  • check credit
  • input_queue.put()
ComputeLoopin StageWorker
  • input_queue.get()
  • BatchPolicy.choose() 组 batch
  • StageExecutor.run_batch()
  • 处理 cancel / flush / close
  • output_queue.put()
SendLoopin StageRuntime
  • output_queue.get()
  • routes_for(output.port) 选路
  • check edge credit / hold / merge / drop
  • transport.write_payload() 搬数据
  • downstream.recv_data_event.remote() 发通知

Section 3

组件职责

五个层次自外向内：StreamRuntimeActor · StageActor · StageRuntime · StageWorker · StageExecutor。后面三个住在同一个 Ray actor 进程里。

3.1 · StreamRuntimeActor — 系统级控制面

负责

  • 创建 Ray placement group 和 StageActor
  • 编译固定 Qwen graph preset
  • 安装每个 stage 的 routing table
  • 维护 SessionState：alive / epoch / turn / SLO / KV lease
  • 接 API 输入，把 input chunk 转成 entry DataEvent
  • 处理 BARGE_IN · CLOSE_SESSION · FLUSH · UPDATE_SLO
  • multicast control event 给相关 stage

不负责

  • 不转发 Thinker → Talker → Code2Wav 的每个 chunk
  • 不持有大 tensor
  • 不替 stage 做 batch scheduling

第一版可以中心化；它不在 payload 热路径上。未来如果成为瓶颈，可按 session_id shard。

@ray.remote class StreamRuntimeActor: async def start_graph(self, graph: GraphSpec) -> RuntimeInfo: ... async def open_session(self, req: OpenSessionRequest) -> SessionInfo: ... async def recv_input_chunk(self, chunk: InputChunk) -> Ack: ... async def recv_control_event(self, event: ControlEvent) -> Ack: ... async def close_session(self, session_id: str, reason: str) -> Ack: ... async def shutdown_runtime(self) -> None: ...

3.2 · StageActor — Ray RPC 边界

一个物理 stage 一个 Ray actor。内部包含 StageRuntime 和 StageWorker。

@ray.remote class StageActor: async def start_stage(self, spec: StageSpec, runtime: RuntimeInfo) -> StageInfo: ... async def set_routes(self, routes: tuple[Route, ...]) -> Ack: ... async def recv_data_event(self, event: DataEvent) -> Ack: ... async def recv_control_event(self, event: ControlEvent) -> Ack: ... async def recv_credit_event(self, event: CreditEvent) -> Ack: ... async def stop_stage(self) -> None: ...
recv_data_event() 只能传 DataEvent + PayloadRef metadata， 不能把 tensor body 放进 Ray object store。 所有 recv_*() 都是 RPC handler，只调用本地 StageRuntime.accept_*() 然后返回 Ack。

3.3 · StageRuntime — streaming runtime shell

每个 stage 的 streaming runtime shell。对应 SGLang-Omni 的 stage runtime，不是 model engine， 不是 Ray RPC actor。Ray RPC 边界在 StageActor 上。

能力方法 / 循环说明
accept accept_data_event
accept_control_event
accept_credit_event		 RPC handler 调用，只记录 metadata 或更新轻量状态，不读大 payload。
RecvLoop recv_loop() 消费 accept 留下的事件；用 TransportManager.read_payload() 解析 payload，过滤 stale event，写入 input_queue。
transport write_payload · read_payload · release_payload 搬数据；worker 输出后存 payload；用完释放。
routing table routes_for(output.port) 根据 worker 输出的 port 选择下游 stage 和 edge。
credit / backpressure check_credit · release_credit 检查 edge credit，决定 data 是否可进 input_queue 或发给下游；没 credit 时 hold / merge / drop。
SendLoop send_loop() / send_output() 从 output_queue 取 worker 输出，按 routing + credit + transport 转成新的 DataEvent，通知下游。

StageRuntime 最小代码形态

@ray.remote class StageActor: async def recv_data_event(self, event: DataEvent) -> Ack: # RPC handler: 只接收通知，不搬 payload，不跑模型。 self.runtime.accept_data_event(event) return Ack.ok() class StageRuntime: def accept_data_event(self, event: DataEvent) -> None: # 本地方法: metadata-only，供 recv_loop 后续消费。 self.recv_buffer.put_nowait(event) async def recv_loop(self) -> None: while True: event = await self.recv_buffer.get() payload = await self.transport.read_payload(event.payload) resolved = ResolvedDataEvent(event=event, payload=payload) await self.input_queue.put(resolved) async def send_output(self, output: SchedulerOutput) -> None: for route in self.routes_for(output.port): # transport: 搬 payload 本体，返回 PayloadRef。 ref = await self.transport.write_payload(output.payload, dst=route.dst_stage) # forwarding: 只把 DataEvent + PayloadRef 通知给下游。 event = DataEvent.from_output(output, route, payload=ref) await self.downstream[route.dst_stage].recv_data_event.remote(event)

3.4 · StageWorker — ComputeLoop + 本地调度

本 stage 的计算组件，内部只有一个 ComputeLoop。第一版不单独拆 StageScheduler。

StageWorker

  • 从 input_queue 取 event
  • 用 local batch policy 组 batch
  • 调 StageExecutor.run_batch()
  • 把 SchedulerOutput 放入 output_queue
  • 处理本 stage 的 local cancel / flush / close

StageExecutor

  • 加载模型权重
  • 管本 stage 的模型对象和 device
  • 执行 prefill / decode / codec batch
  • 产出 typed output

第一版可以先 fake executor，先验证 runtime · transport · control · credit。

transport版本载荷类型用途
inlinev1小 CPU 对象直接放 DataEvent handle，无复制

Section 8

Duplex Session 语义

SessionState 是 StreamRuntime 的核心。barge-in / close 通过 control multicast 实现，stage 本地按 epoch 过滤。

@dataclass class SessionState: session_id: str graph_id: str status: str # open · draining · closed active_epoch: int active_turn_id: int slo_class: str kv_leases: dict[str, PayloadRef] stages_holding_state: set[str]

8.1 · 正常输入

1

API server → StreamRuntime

客户端发来 AudioChunk / TextChunk / VideoFrame。

await StreamRuntimeActor.recv_input_chunk(chunk)
2

StreamRuntime 生成 entry DataEvent

更新 session state，挂上当前 active_epoch / active_turn_id。

DataEvent(session, epoch, ..., payload=PayloadRef)
3

entry StageActor.recv_data_event()

RPC handler 接通知，accept_data_event() 入 recv_buffer，立即返回 Ack。

4

后续 stage 间 transport + forwarding

每条 edge 都是 transport 搬 payload + forwarding 发 DataEvent；不再回到 StreamRuntime。

Section 9

Backpressure 与 Deadline

每条 edge 有 bounded credit。downstream 接收并释放后才把 credit 返还给上游 SendLoop。

图 9.1 · credit 在 edge 上的循环
sequenceDiagram participant USL as Upstream SendLoop participant DRT as Downstream StageRuntime participant DCL as Downstream ComputeLoop Note over USL: credit > 0 才允许 forward USL->>DRT: recv_data_event(DataEvent + PayloadRef) DRT-->>USL: Ack + return event credit DCL->>DCL: run_batch · release_payload DCL-->>USL: recv_credit_event(byte credit) Note over USL: SendLoop wake up
继续 forward

9.1 · 规则

  • downstream accepted event 后返回 event credit。
  • payload release 后返回 byte credit。
  • producer 没 credit 时不能继续 forwarding data notification。
  • control event 可以越过 credit，但必须小。
  • stale epoch data 直接 drop 并 release payload。
  • deadline policy 在 RecvLoop 写 input_queue 前和 StageWorker.build_batch() 中都可以使用。

StreamRuntime owns session truth.

Ray owns lifecycle / control · not data body.
