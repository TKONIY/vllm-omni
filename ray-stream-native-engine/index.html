Section 1

三个 plane · 各管一摊

系统拆成三个 plane：control plane（Ray RPC） 启动 actor 和分发 graph； event plane（ZMQ） 在 stage 之间发 DataEvent 通知； data plane（Relay） 搬 payload 本体。三个 plane 各走各的路。

CONTROL

Ray RPC

低频 · graph / session / lifecycle

  • start_graph · open_session
  • recv_input_chunk（API ingress）
  • start_stage · set_routes · stop_stage
  • 不承担 stage-to-stage data
EVENT

ZMQ PUSH / PULL

高频 · stage-to-stage notification

  • 每个 stage bind ipc://.../stage_<id>.sock
  • 上游 PUSH，下游 PULL
  • wire 上只放 DataEvent（pickle）
  • 不携带 payload body
DATA

RelayManager

payload body 真实存储 / 读取 / 释放

  • put(payload) -> PayloadRef
  • get(ref) -> payload
  • release(ref)
  • v1 后端：inline · shm

Section 2

Remote / ZMQ / Local 边界

看清三类方法：标了 @rpc_method 的是 Ray RPC surface； ZMQ surface 只搬 DataEvent；其它都是同一进程内的本地方法。

Ray RPC

@rpc_method

跨进程 / 跨 actor · 低频

  • StreamRuntimeActor：start_graph · open_session · recv_input_chunk · close_session · get_stage_outputs · shutdown_runtime
  • StageActor：start_stage · set_routes · get_sink_outputs · get_stage_info · stop_stage
  • StageActor 不提供 data / control / credit receive RPC
ZMQ surface

DataEvent · 唯一通过

stage-to-stage event plane

  • wire 上只有 DataEvent 一种结构（pickle）
  • 不带 route 字段：目的 stage 由 ZMQ endpoint 决定
  • 不带 src 字段：源 stage 是当前发送方 runtime
  • metadata 只塞 modality / seq_no / edge_id 这类附加信息
  • 不塞 scheduling / control 协议
Local-only

普通 Python 方法

同 actor 进程内 · 不暴露到 Ray

  • StageRuntime.recv_loop · send_loop · handle_event
  • StageWorker.compute_loop
  • RelayManager.put · get · release
  • StageExecutor.load_model · run_batch · unload_model

三条 Loop 的分工

RecvLoopin StageRuntime
  • event_receiver.recv_event()（ZMQ PULL）
  • relay.get(event.payload)
  • relay.release(event.payload)
  • input_queue.put(StageInput)
ComputeLoopin StageWorker
  • input_queue.get()
  • executor.run_batch([item])
  • output_queue.put(StageOutput)
  • （暂无 batch policy / EDF / credit 检查）
SendLoopin StageRuntime
  • output_queue.get()
  • routes_by_port[output.port] 选路
  • relay.put(output.payload)
  • event_sender.send_event(endpoint, event)
  • 无 route 时 fallback 写 sink_outputs（debug 用）
Cheat sheet。 凡是被 @rpc_method 标记的 → Ray RPC； 凡是出现 ZMQ socket 的 → 是 event plane； 其它一律是本进程的 asyncio task / async method。
Section 3

五层组件

外到内：StreamRuntimeActor · StageActor · StageRuntime · StageWorker · StageExecutor。前两个是 Ray actor，后三个住在同一个 Ray actor 进程里。

3.1 · StreamRuntimeActor — 全局入口

Ray actor，graph 级别的入口。它负责分配 IPC 路径、创建 StageActor、下发 routes， 并且为每条 API ingress chunk 跑一次 relay.put + zmq send，把第一条 DataEvent 推给 ingress stage。

@ray.remote(max_concurrency=128) class StreamRuntimeActor: def __init__(self): self.graph: GraphSpec | None = None self.stage_actors: dict[str, Any] = {} self.stage_endpoints: dict[str, str] = {} self.ipc_runtime_dir: str | None = None self.sessions: dict[str, SessionState] = {} self.event_sender = ZmqEventSender() self.relay = RelayManager(owner_stage="stream_runtime") @rpc_method async def start_graph(self, graph: GraphSpec) -> RuntimeInfo: ... @rpc_method async def open_session(self, req: OpenSessionRequest) -> SessionInfo: ... @rpc_method async def recv_input_chunk(self, chunk: InputChunk) -> Ack: ... @rpc_method async def close_session(self, session_id: str, reason: str = "") -> Ack: ... @rpc_method async def get_stage_outputs(self, stage_id: str) -> list: ... @rpc_method async def shutdown_runtime(self) -> None: ...

start_graph 做的事

  • prepare_ipc_runtime_dir(graph_id) 在 /tmp/omnistream/<graph_id> 准备一个 deterministic 目录。
  • 给每个 stage allocate_ipc_endpoint(runtime_dir, stage_id)，得到 ipc://.../stage_<id>.sock。
  • 逐个 StageActor.options(num_cpus=0).remote() + await actor.start_stage.remote(spec, runtime_info, endpoint)。
  • 按 graph.routes 计算每个 src stage 的 routes 和 downstream endpoint map，await actor.set_routes.remote(routes, endpoints)。

recv_input_chunk 做的事

  • 按 chunk.modality 从 graph.ingress 查 entry stage。
  • ref = await self.relay.put(chunk.payload)。
  • 构造 DataEvent(session_id, payload=ref, metadata={…, "modality": modality})。
  • await self.event_sender.send_event(endpoint, event) 用 ZMQ PUSH 把 event 推给 ingress stage。

3.2 · StageActor — Ray RPC lifecycle wrapper

每个物理 stage 一个 Ray actor，只是 lifecycle wrapper。它持有一个本地的 StageRuntime，所有 data / event 都不通过它的 Ray RPC 表面进出。

@ray.remote(max_concurrency=128) class StageActor: def __init__(self): self.runtime: StageRuntime | None = None @rpc_method async def start_stage( self, spec: StageSpec, runtime: RuntimeInfo, control_endpoint: str, ) -> StageInfo: ... @rpc_method async def set_routes( self, routes: tuple[Route, ...], downstream_endpoints: dict[str, str], ) -> Ack: ... @rpc_method async def get_sink_outputs(self) -> list[StageOutput]: ... @rpc_method async def get_stage_info(self) -> StageInfo | None: ... @rpc_method async def stop_stage(self) -> None: ...
注意。 StageActor 没有 recv_data_event() / recv_control_event() / recv_credit_event() 这类 Ray RPC handler。
所有 stage-to-stage 通知必须走 ZMQ； get_sink_outputs 只是 demo / debug 通道，下一步会被 terminal output channel 替掉。

3.3 · StageRuntime — 本地运行时

StageActor 内部的本地运行时，不是 Ray actor。 持有 ZMQ receiver / sender、relay、两个 queue、routing table 和三条 asyncio loop。

能力方法 / 字段说明
recv recv_loop() · handle_event() 从 ZmqEventReceiver 拉 DataEvent，调 relay.get、relay.release，写 input_queue。
relay relay: RelayManager 本 stage 的数据面后端（inline + shm）。
queues input_queue · output_queue 本 stage 的两条主队列；都是 unbounded asyncio.Queue（v1 不做 credit）。
routing routes_by_port · downstream_endpoints set_routes() 一次写入；按 StageOutput.port 选下游 stage 和 ZMQ endpoint。
send send_loop() · send_output() 从 output_queue 取，relay.put 后用 ZmqEventSender 发给下游。
debug sink_outputs: list 对没有 route 的 stage（terminal），把 StageOutput 追加进来，供 get_sink_outputs 读出。

StageRuntime 最小代码形态

class StageRuntime: async def recv_loop(self) -> None: while not self._stopping.is_set(): event = await self.event_receiver.recv_event() await self.handle_event(event) async def handle_event(self, event: DataEvent) -> None: payload = await self.relay.get(event.payload) await self.relay.release(event.payload) await self.input_queue.put( StageInput(session_id=event.session_id, payload=payload, metadata=dict(event.metadata)) ) async def send_loop(self) -> None: while not self._stopping.is_set(): output = await self.output_queue.get() await self.send_output(output) async def send_output(self, output: StageOutput) -> None: routes = self.routes_by_port.get(output.port, []) if not routes: self.sink_outputs.append(output) return for route in routes: ref = await self.relay.put(output.payload) event = DataEvent( session_id=output.session_id, payload=ref, metadata={**output.metadata, "edge_id": route.edge_id}, ) endpoint = self.downstream_endpoints[route.dst_stage] await self.event_sender.send_event(endpoint, event)

3.4 · StageWorker — ComputeLoop

本 stage 的计算组件，内部只有一个 compute_loop。第一版每次只跑一个 item，没有 batch policy。

StageWorker

  • 从 input_queue 取一个 StageInput
  • await self.executor.run_batch([item])
  • 把 StageOutput 逐个塞进 output_queue
  • 停止信号是 _stopping: asyncio.Event

StageExecutor

  • load_model(spec) / unload_model()
  • run_batch(batch: list[StageInput]) -> list[StageOutput]
  • v1 是 FakeStageExecutor，把 path append 上 stage_id
  • 下一步：Qwen preprocessing / encoder / thinker / talker / codec adapter
class StageWorker: async def compute_loop(self) -> None: while not self._stopping.is_set(): item = await self.input_queue.get() outputs = await self.executor.run_batch([item]) for output in outputs: await self.output_queue.put(output) class StageExecutor(ABC): async def load_model(self, spec: StageSpec) -> None: ... async def run_batch(self, batch: list[StageInput]) -> list[StageOutput]: ... async def unload_model(self) -> None: ...

Section 4

事件结构

v1 只有一种事件：DataEvent。 没有 ControlEvent、没有 CreditEvent、没有 BARGE_IN 多播 — 这些等 minimal serve 跑通后再加。

4.1 · DataEvent

@dataclass(frozen=True) class DataEvent: session_id: str payload: PayloadRef # 只是 handle，不是 body metadata: dict[str, Any] = field(default_factory=dict)

没有 route 字段：目的 stage 已经由 ZMQ endpoint 决定，源 stage 由当前发送方 runtime 知道。 metadata 只临时放 modality / seq_no / edge_id， 不塞 scheduling / control 协议。

4.2 · PayloadRef

@dataclass(frozen=True) class PayloadRef: payload_id: str kind: str # data / hidden / audio / ... device: str # cpu / cuda dtype: str | None shape: tuple[int, ...] | None nbytes: int relay: str # inline / shm / (cuda_ipc / kv_lease / nixl v2) handle: Any # inline: payload itself; shm: {name, size} owner_stage: str lifetime: str = "event"

4.3 · 边界数据

InputChunk

API server → StreamRuntimeActor

  • session_id
  • modality（决定 entry stage）
  • payload
  • metadata

StageInput

RecvLoop → ComputeLoop

  • session_id
  • payload（已 relay.get 出来）
  • metadata

StageOutput

ComputeLoop → SendLoop

  • session_id
  • port（routing key）
  • payload

Section 5

ZMQ 事件面

每个 stage 一个 PULL endpoint；上游用 PUSH 连过去发 DataEvent。 endpoint 由 StreamRuntime 在 start_graph 时统一分配，stage 不自己挑。

5.1 · IPC endpoint

runtime_dir = /tmp/omnistream/<safe_graph_id> stage endpoint = ipc://<runtime_dir>/stage_<safe_stage_id>.sock

prepare_ipc_runtime_dir(graph_id) 会 rmtree 同名旧目录后重建， allocate_ipc_endpoint(runtime_dir, stage_id) 把不安全字符 sanitize 成 [0-9A-Za-z_.-]。 cleanup_ipc_runtime_dir(...) 在 shutdown_runtime 时擦掉整个目录。

5.2 · ZmqEventReceiver / ZmqEventSender

ZmqEventReceiver

stage-local · PULL · bind

  • start()：mkdir 父目录、unlink 旧 socket、socket.bind(endpoint)
  • recv_event()：await socket.recv() → pickle.loads
  • close()：socket.close(linger=0)
  • 共享同一个 zmq.asyncio.Context.instance()

ZmqEventSender

lazy keyed by destination endpoint · PUSH · connect

  • send_event(endpoint, event)：缺 socket 就新建 + connect
  • pickle.dumps(event, HIGHEST_PROTOCOL)
  • close()：所有 socket 一起 close
  • StreamRuntimeActor 和每个 StageRuntime 各持有一份

5.3 · wire 上有什么

消息序列化方向说明
DataEventpickle上游 PUSH → 下游 PULL唯一的 ZMQ payload；带 PayloadRef
———v1 没有别的 wire message：ControlType / CreditEvent / heartbeat 都不在面上
为什么 ZMQ 而不是 Ray RPC 搬 DataEvent？ Ray actor 之间的 .remote() 会经过 Ray RPC 栈、序列化以及 actor task queue； 高频小消息会被淹没。改用 ZMQ PUSH/PULL 之后 wire 直接落 IPC socket， 没有 Ray object store 命中，调度延迟也不被 Ray actor 的 max_concurrency 卡。
Section 6

Relay · 数据面

ZMQ 不搬 body，RelayManager 搬。v1 只支持 inline 和 shm； cuda_ipc / kv_lease / nixl 留给 v2。

6.1 · 接口

class RelayManager: def __init__(self, owner_stage: str, inline_threshold_bytes: int = 64 * 1024): ... async def put(self, payload: Any, *, kind: str = "data") -> PayloadRef: ... async def get(self, ref: PayloadRef) -> Any: ... async def release(self, ref: PayloadRef) -> None: ...

6.2 · 选 backend 的规则

inline

小 CPU 对象 · 默认

  • payload 不是 bytes/bytearray/memoryview，或大小 ≤ 64 KiB
  • handle = payload 本身
  • get() 返回 handle，release() noop

shm

大 CPU bytes · multiprocessing.shared_memory

  • payload 是 bytes/bytearray/memoryview 且大小 > 64 KiB
  • handle = {"name": shm.name, "size": nbytes}
  • get() 重 attach 读出 bytes 并 close
  • release() 调 shm.unlink() + close()，幂等

6.3 · 后端矩阵

relay版本载荷类型用途
inlinev1小 CPU 对象 / 任意 Python objectPayloadRef.handle 就是对象本体
shmv1> 64 KiB CPU bytes / bytearray / memoryview跨进程共享，不复制
cuda_ipcv2same-node GPU hidden / codec tensorsame-node Ray actor 间共享显存
kv_leasev2KV cache传 lease id / block id，不复制 KV
nixlv2cross-node GPU / CPU跨节点高速通路

6.4 · 写入 / 读取路径

写入（上游 SendLoop）

  1. ref = await relay.put(payload)
  2. event = DataEvent(session_id, payload=ref, metadata)
  3. await event_sender.send_event(endpoint, event)

读取（下游 RecvLoop）

  1. event = await event_receiver.recv_event()
  2. payload = await relay.get(event.payload)
  3. await relay.release(event.payload)
  4. await input_queue.put(StageInput(...))
注意 v1 的 release 语义。 handle_event 拿到 payload 后就立刻 release；shm 后端会 unlink 共享内存。 也就是说每个 payload 只被消费一次。当后续支持 fan-out 时，需要把 release 改成 引用计数 / lifetime 管理，不能再这样立刻 unlink。

8.1 · 文字版分解

1

API client → StreamRuntimeActor

客户端通过 Ray RPC 把 chunk 推进来。

await runtime.recv_input_chunk.remote(InputChunk(session_id, modality, payload, metadata))
2

StreamRuntime relay.put + build DataEvent

查 session（不存在则自动 open），按 modality 取 ingress stage 的 ZMQ endpoint。

ref = await self.relay.put(chunk.payload) · DataEvent(session_id, payload=ref, metadata={..., "modality": modality})
3

Section 9

Queues 与三条 Loop

每个 StageRuntime 里只有两条主队列。ZMQ socket 本身充当上游 buffer， 所以没有 recv_buffer。

图 9.1 · 两个 queue + 三条 loop
flowchart LR classDef zmq fill:#F4E2D7,stroke:#A85940,color:#1F1E1C classDef store fill:#E0E8DC,stroke:#6E8B6A,color:#1F1E1C classDef loop fill:#FAF7F0,stroke:#CC785C,color:#1F1E1C ZRX[ZMQ PULL socket]:::zmq --> RL[recv_loop]:::loop RL -- "relay.get + release" --> IQ[(input_queue)]:::store IQ --> CL[compute_loop]:::loop CL -- "executor.run_batch" --> OQ[(output_queue)]:::store OQ --> SL[send_loop]:::loop SL -- "relay.put · zmq send" --> ZTX[ZMQ PUSH socket]:::zmq
名字谁写谁读含 payload body？承载语义
input_queue recv_loop / handle_event compute_loop 是 · StageInput.payload 已 relay.get 出来 executor 的输入
output_queue compute_loop send_loop 是 · StageOutput.payload 跨 stage forwarding 的待发队列
sink_outputs send_output（无 route 时） get_sink_outputs（Ray RPC） 是 terminal stage 临时调试通道
队列都是 unbounded。 v1 故意不做 credit / backpressure。这意味着如果下游 stage 落后， output_queue、input_queue、以及 ZMQ 的 HWM 都会 build up。 Qwen3-Omni minimal serve 跑通之后再决定是否补 credit（见 Section 10）。
Section 10

  • ControlType / ControlEvent · 没有 BARGE_IN / CANCEL_EPOCH / FLUSH
