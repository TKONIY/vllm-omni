OmniStream
Ray · ZMQ · Relay 架构

Research prototype，目标是先把 Qwen3-Omni 类多 stage 模型 serve 跑起来。 Ray 只负责启动和生命周期；ZMQ control plane 负责小消息；Relay data plane 负责真实 payload； StageExecutor 负责模型计算。 第一阶段不兼容 vLLM engine 接口，不引入 control / credit / backpressure 协议。

Section 1

三个 plane · 各管一摊

系统拆成三个 plane：control plane（Ray RPC） 启动 actor 和分发 graph； event plane（ZMQ TCP） 在 stage 之间发 ControlPlaneDataEvent 通知； data plane（Relay） 搬 payload 本体。三个 plane 各走各的路。

LIFECYCLE

Ray RPC

低频 · graph / session / actor lifecycle

  • StreamActor: start_graph · recv_input_chunk · open_session · ...
  • StageActor: start_stage · stop_stage · get_sink_outputs
  • 不承担 stage-to-stage data
CONTROL PLANE

StageControlPlane / RuntimeControlPlane

封装 ZMQ TCP PUSH/PULL · 高频 stage 通知

  • 底层 PullSocket（TCP bind）/ PushSocket（TCP connect）
  • 每个 stage 的 endpoint 在 StageControlSpec 里 declare
  • wire 上只放 ControlPlaneDataEvent（pickle）
  • 不携带 payload body
DATA PLANE

Relay · LocalRelay

SGLang-style · payload 存取与生命周期

  • put_async / get_async / cleanup / close
  • RelayMetadata 是 ABC，每个 relay 自己 declare subtype
  • v1 实现：LocalRelay（inline · shm）
  • v2：nixl · mooncake · cuda_ipc

六层调用栈

从 API client 到 GPU kernel，OmniStream 一共 6 层。前两层和第 3 层都是 Ray actor + 本地 runtime 的 两两组合：actor 只是 RPC 边界，runtime 才是真正干活的对象。

为什么有两套 actor+runtime split？ StreamActor 和 StageActor 都是 @ray.remote； 它们只是 Ray RPC 的边界，里面持有真正的本地 StreamRuntime / StageRuntime。 这样可以：① 把 RPC handler 写得很薄（一行 return await self.runtime.foo(...)）， ② 让 runtime 单独跑单元测试不需要 Ray，③ 以后把 RPC 接口换成 HTTP / gRPC 也不影响 runtime。
Section 3

Remote / ZMQ / Local 边界

看清三类方法：标了 @rpc_method 的是 Ray RPC surface； ZMQ surface 只搬 ControlPlaneDataEvent；其它都是同一进程内的本地方法。

Ray RPC

@rpc_method

跨进程 / 跨 actor · 低频

  • StreamActor: start_graph · shutdown_runtime · open_session · recv_input_chunk · close_session · get_stage_outputs
  • StageActor: start_stage · get_sink_outputs · stop_stage
  • StageActor 不提供 data / control / credit receive RPC
ZMQ surface

ControlPlaneDataEvent · 唯一通过

stage-to-stage event plane

  • wire 上只有 ControlPlaneDataEvent（pickle, TCP）
  • 不带 route 字段：目的 stage 由 endpoint 决定
  • 不带 src 字段：源 stage 是当前发送方
  • chunk_metadata 只塞 modality / seq_no / edge_id
  • relay_metadata 是 relay 后端返回的 opaque handle
Local-only

普通 Python 方法

同 actor 进程内 · 不暴露到 Ray

  • StreamRuntime.*（StreamActor 的真实实现）
  • StageRuntime._recv_loop · _send_loop · _handle_event · _send_output
  • StageWorker.compute_loop
  • Relay.put_async · get_async · cleanup · close
  • RelayOperation.wait_for_completion
  • StageExecutor.load_model · run_batch · unload_model

三条 Loop 的分工

_recv_loopin StageRuntime
  • control.recv()（ZMQ PULL）
  • op = await relay.get_async(event.relay_metadata, request_id=event.session_id)
  • await op.wait_for_completion()
  • relay.cleanup(event.session_id)
  • input_queue.put(StageExecutorInput)
compute_loopin StageWorker
  • input_queue.get()
  • executor.run_batch([item])
  • output_queue.put(StageExecutorOutput)
  • （暂无 batch policy / EDF / credit 检查）
_send_loopin StageRuntime
  • output_queue.get()
  • routes_by_port[output.port] 选路
  • op = await relay.put_async(output.payload, request_id=output.session_id)
  • control.send_to_stage(dst, stage_specs[dst].control.connect_endpoint, event)
  • await op.wait_for_completion()
  • 无 route 时 fallback 写 sink_outputs（debug）
Cheat sheet。 被 @rpc_method 标 → Ray RPC； 出现 ZMQ socket → event plane； 其它一律本进程 asyncio task / async method。
Section 4

组件细节

逐层 ×（actor / runtime）的真实接口。所有签名都对应 ~/code/omnistream 当前源码。

4.1 · StreamActor — Ray RPC 边界

thin Ray actor，max_concurrency=1，所有 @rpc_method 一行 delegate 到内部 StreamRuntime。

@ray.remote(max_concurrency=1) class StreamActor: def __init__(self): self.runtime = StreamRuntime() @rpc_method async def start_graph(self, graph: GraphSpec) -> RpcAck: return await self.runtime.start_graph(graph) @rpc_method async def shutdown_runtime(self) -> None: await self.runtime.shutdown() # API entrypoints (delegate to runtime) @rpc_method async def open_session(self, req: StreamRuntimeOpenSessionRequest) -> StreamRuntimeSessionInfo: ... @rpc_method async def recv_input_chunk(self, chunk: StreamRuntimeInputChunk) -> RpcAck: ... @rpc_method async def close_session(self, session_id: str, reason: str = "") -> RpcAck: ... @rpc_method async def get_stage_outputs(self, stage_id: str) -> list: ...

4.2 · StreamRuntime — graph / session / API ingress 真理

不是 Ray actor；普通 class。持有所有 StageActor 的 ray handle、自己的 RuntimeControlPlane、自己的 LocalRelay，以及 session table。

class StreamRuntime: def __init__(self): self.graph: GraphSpec | None = None self.stage_actors: dict[str, Any] = {} self.sessions: dict[str, StreamRuntimeSessionState] = {} self.control = RuntimeControlPlane() self.relay = LocalRelay(owner_stage="stream_runtime") async def start_graph(self, graph: GraphSpec) -> RpcAck: ... async def shutdown(self) -> None: ... async def open_session(self, req) -> StreamRuntimeSessionInfo: ... async def recv_input_chunk(self, chunk: StreamRuntimeInputChunk) -> RpcAck: ... async def close_session(self, session_id, reason="") -> RpcAck: ... async def get_stage_outputs(self, stage_id: str) -> list: ...

start_graph 做的事

  • 从 graph.routes 编译每个 src stage 的 routes tuple。
  • 对每个 stage StageActor.options(num_cpus=0).remote() 创建一个 Ray actor。
  • 用 asyncio.gather(*[actor.start_stage.remote(spec, routes, stage_specs) for ...]) 并行启动所有 stage。
    注意：stage_specs（全量 stage 字典）一并传给每个 stage，方便 stage 自己查下游 control endpoint。

recv_input_chunk 做的事

  • 按 chunk.modality 从 graph.ingress 查 entry stage。
  • op = await self.relay.put_async(chunk.payload, request_id=chunk.session_id)。
  • 构造 ControlPlaneDataEvent(session_id, relay_metadata=op.metadata, chunk_metadata={..., "modality": modality})。
  • await self.control.send_to_stage(entry_stage, entry_spec.control.connect_endpoint, event)。
  • await op.wait_for_completion()（v1 是 no-op，但保留接口）。

4.3 · StageActor — Ray RPC lifecycle wrapper

每个物理 stage 一个 Ray actor。RPC 表面已经收窄到只剩三个。

@ray.remote(max_concurrency=128) class StageActor: def __init__(self): self.runtime: StageRuntime | None = None @rpc_method async def start_stage( self, spec: StageSpec, routes: tuple[Route, ...], stage_specs: dict[str, StageSpec], ) -> RpcAck: ... @rpc_method async def get_sink_outputs(self) -> list[StageExecutorOutput]: ... @rpc_method async def stop_stage(self) -> None: ...
注意。 ① 没有 set_routes RPC 了 — routes 在 start_stage 时一次传完；
② 没有 get_stage_info RPC 了；
③ 没有任何 recv_data_event / recv_control_event Ray RPC handler — stage 间通信只走 ZMQ。

4.4 · StageRuntime — 本 stage 真理源

持有 StageControlPlane（PULL bind + 下游 PUSH cache）、LocalRelay、 两个 queue、routes_by_port 路由表、对全量 stage_specs 的视图（为了查下游 endpoint）和三条 asyncio loop。

能力方法 / 字段说明
recv _recv_loop() · _handle_event() control.recv() 拉 event；调 relay.get_async + wait_for_completion + relay.cleanup(session_id)；写 input_queue。
relay relay: LocalRelay 本 stage 自己的 relay 实例；按 request_id 跟踪 ref 生命周期。
queues input_queue · output_queue unbounded asyncio.Queue。
routing routes_by_port · stage_specs output port → routes 表；下游 endpoint 通过 stage_specs[dst].control.connect_endpoint 查。
send _send_loop() · _send_output() 从 output_queue 取，relay.put_async 后用 control.send_to_stage 发给下游。
debug sink_outputs: list 无 route 时把 StageExecutorOutput 追加进来，供 get_sink_outputs 读出。

StageRuntime 最小代码形态

class StageRuntime: async def _recv_loop(self) -> None: while not self._stopping.is_set(): event = await self.control.recv() await self._handle_event(event) async def _handle_event(self, event: ControlPlaneDataEvent) -> None: op = await self.relay.get_async(event.relay_metadata, request_id=event.session_id) await op.wait_for_completion() payload = op.payload self.relay.cleanup(event.session_id) await self.input_queue.put( StageExecutorInput( session_id=event.session_id, payload=payload, chunk_metadata=dict(event.chunk_metadata), ) ) async def _send_loop(self) -> None: while not self._stopping.is_set(): output = await self.output_queue.get() await self._send_output(output) async def _send_output(self, output: StageExecutorOutput) -> None: routes = self.routes_by_port.get(output.port, []) if not routes: self.sink_outputs.append(output) return for route in routes: op = await self.relay.put_async(output.payload, request_id=output.session_id) event = ControlPlaneDataEvent( session_id=output.session_id, relay_metadata=op.metadata, chunk_metadata={**output.chunk_metadata, "edge_id": route.edge_id}, ) endpoint = self.stage_specs[route.dst_stage].control.connect_endpoint await self.control.send_to_stage(route.dst_stage, endpoint, event) await op.wait_for_completion()

4.5 · StageWorker — compute_loop

本 stage 的计算组件，内部只有一个 compute_loop。第一版每次只跑一个 item，没有 batch policy。

class StageWorker: async def compute_loop(self) -> None: while not self._stopping.is_set(): item = await self.input_queue.get() outputs = await self.executor.run_batch([item]) for output in outputs: await self.output_queue.put(output)

Section 5

事件结构

v1 只有一种 stage-to-stage 事件：ControlPlaneDataEvent。 没有 ControlEvent、没有 CreditEvent、没有 BARGE_IN 多播 — 这些等 minimal serve 跑通后再考虑。

5.1 · ControlPlaneDataEvent

@dataclass(frozen=True) class ControlPlaneDataEvent: session_id: str relay_metadata: RelayMetadata # ABC; LocalRelay 下是 LocalRelayMetadata chunk_metadata: dict[str, Any] = field(default_factory=dict)

没有 route 字段：目的 stage 已经由 ZMQ endpoint 决定，源 stage 是当前发送方。 relay_metadata 是 Relay 后端返回的 handle，下游用它再 relay.get_async 拿真实 payload。 chunk_metadata 只塞 modality / seq_no / edge_id， 不塞 scheduling / control / SLO / deadline 协议。

5.2 · 边界数据

StreamRuntimeInputChunk

API client → StreamActor

  • session_id
  • modality（决定 entry stage）
  • payload
  • chunk_metadata

StageExecutorInput

_recv_loop → compute_loop

  • session_id
  • payload（已 relay.get_async 出来）
  • chunk_metadata

StageExecutorOutput

compute_loop → _send_loop

  • session_id
  • port（routing key）
  • payload
  • chunk_metadata

Section 6

Control Plane · TCP ZMQ

v4 起把 IPC 换成 TCP，让 control plane 可以跨节点。 每个 stage 在 StageControlSpec 里 declare 自己的 listen_host:port 和 connect_host:port。 runtime 和上游 stage 用 connect_endpoint 连过来。

6.1 · StageControlSpec

@dataclass(frozen=True) class StageControlSpec: listen_host: str listen_port: int connect_host: str connect_port: int @property def listen_endpoint(self) -> str: return f"tcp://{self.listen_host}:{self.listen_port}" @property def connect_endpoint(self) -> str: return f"tcp://{self.connect_host}:{self.connect_port}"

典型多机配置：

StageActor on node N listen: tcp://0.0.0.0:19000 # PullSocket bind connect: tcp://10.1.2.3:19000 # peers connect here

6.2 · PullSocket / PushSocket（低层）

PullSocket

stage-local · PULL · TCP bind

  • 构造时传 StageControlSpec，从中取 listen_endpoint 做 bind
  • connect_endpoint 暴露给外部，告诉上游往哪连
  • start() · recv() → pickle.loads → ControlPlaneDataEvent
  • close()
  • 共享 zmq.asyncio.Context.instance()

PushSocket

PUSH · TCP connect 单一 endpoint

  • 构造时传 endpoint: str
  • connect()：socket.connect(endpoint)
  • send(event)：pickle.dumps(event, HIGHEST_PROTOCOL)
  • close()

6.3 · StageControlPlane / RuntimeControlPlane（上层）

StageControlPlane

每个 stage 各一份 · PULL bind + PUSH cache

  • __init__(stage_id, *, control: StageControlSpec)
  • start() → recv() 拉 ControlPlaneDataEvent
  • send_to_stage(stage_id, endpoint, event)：缺 PushSocket 就新建 connect，缓存在 _stage_sockets[stage_id]
  • connect_endpoint 属性暴露本 stage 的连接地址
  • close() 关 PULL + 全部下游 PUSH socket

RuntimeControlPlane

StreamRuntime 一份 · 只 PUSH，没有 PULL

  • 没有 recv 路径：API ingress 走 Ray RPC，不走 ZMQ
  • send_to_stage(stage_id, endpoint, event)
  • close()
  • 只在 recv_input_chunk 里 PUSH 一次 entry event

6.4 · wire 上有什么

消息序列化方向说明
ControlPlaneDataEventpickle (HIGHEST_PROTOCOL)上游 TCP PUSH → 下游 TCP PULL唯一 ZMQ payload；带 RelayMetadata
———v1 没有别的 wire message：ControlType / CreditEvent / heartbeat 都不在面上
安全。 Control message 用 pickle 序列化，只能跑在受信任内网； 暴露到不可信网络前需要加认证或换安全序列化格式。
跨节点能力。 Control plane 现在 TCP，理论上能跨机器。 但 payload 能否跨机器还要看 relay backend — 当前 LocalRelay 的 shm 模式仍然只适合同机； 跨节点 payload 需要后面接 nixl / mooncake / nccl relay。
Section 7

Relay · 数据面

SGLang-style：put / get 返回 RelayOperation，按 request_id 管理 ref 生命周期。 v4 把代码拆成 relay/base.py（接口）和 relay/local.py（v1 实现）。 RelayMetadata 是 ABC，每个 relay 自己 declare metadata subtype。

7.1 · 模块拆分

文件内容

relay/local.pyLocalRelayMetadata（concrete dataclass）· LocalRelay（inline + shm 实现）

7.3 · LocalRelayMetadata

@dataclass(frozen=True) class LocalRelayMetadata(RelayMetadata): payload_id: str kind: str device: str dtype: str | None shape: tuple[int, ...] | None nbytes: int relay: str # inline / shm handle: Any # inline: payload itself; shm: {name, size} owner_stage: str lifetime: str = "event"

7.4 · LocalRelay v1 backend 规则

inline

小 CPU 对象 · 默认

  • payload 不是 bytes/bytearray/memoryview，或大小 ≤ 64 KiB
  • handle = payload 本身
  • get_async() 返回 RelayOperation(metadata=ref, payload=ref.handle)
  • cleanup() 对 inline 是 no-op

shm

大 CPU bytes · multiprocessing.shared_memory

  • payload 是 bytes/bytearray/memoryview 且大小 > 64 KiB
  • handle = {"name": shm.name, "size": nbytes}
  • get_async() 重 attach 读出 bytes 并 close
  • cleanup() 调 shm.unlink() + close()，幂等

7.5 · ref 生命周期与 request_id

LocalRelay._refs_by_key: dict[str, list[LocalRelayMetadata]]： put / get 都把 ref 同时挂到 payload_id 和 request_id 两个 key 上。 cleanup(request_id) 把 key 下所有 ref 释放（shm unlink），并从 payload_id 索引里摘掉。

当前 prototype 把 request_id 用作 session_id。 _handle_event 一拿到 payload 就 cleanup(event.session_id)； _send_output 用 output.session_id 作 put 的 request_id。 这种 1-event = 1-cleanup 适合 v1 单消费 / 无 fan-out； fan-out 或多消费场景需要重新设计（例如改用 turn_id / chunk_id 作 request_id，或加 refcount）。

7.7 · 写入 / 读取路径

写入（上游 _send_output）

  1. op = await relay.put_async(payload, request_id=session_id)
  2. event = ControlPlaneDataEvent(session_id, relay_metadata=op.metadata, chunk_metadata)
  3. endpoint = stage_specs[dst].control.connect_endpoint
  4. await control.send_to_stage(dst, endpoint, event)
  5. await op.wait_for_completion()

读取（下游 _handle_event）

  1. event = await control.recv()
  2. op = await relay.get_async(event.relay_metadata, request_id=event.session_id)
  3. await op.wait_for_completion()
  4. relay.cleanup(event.session_id)
  5. await input_queue.put(StageExecutorInput(...))
